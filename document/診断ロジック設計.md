# Boardory 診断ロジック設計

## 概要

Boardoryは「ボードゲームの嗜好からユーザーのプレイスタイルを可視化する」サービスである。

本ドキュメントでは、BGGデータ（mechanics / categories / weight）を利用した診断ロジックを定義する。

---

# 1. 設計方針

## 1.1 基本方針

- 機械学習は使用しない
- 完全ルールベースで実装する
- 精度より「納得感」と「一貫性」を優先する

## 1.2 処理フロー


ゲーム選択
↓
BGGデータ取得（mechanics / category / weight）
↓
mechanics → 中間カテゴリ変換
↓
カテゴリスコア集計
↓
Boardoryスコアへ変換
↓
タイプ分類


---

# 2. データソース

## 2.1 BGG API

- XML API2 を利用
- `link[type=boardgamemechanic]` でmechanics取得
- `link[type=boardgamecategory]` でcategory取得

## 2.2 利用フィールド

| フィールド | 用途 |
|-----------|------|
| mechanics | ゲームの特徴抽出 |
| categories | 補助補正 |
| weight | 複雑さ（戦略性補正） |

---

# 3. 中間カテゴリ設計

BGG mechanicsを直接扱わず、以下のカテゴリに圧縮する。

## 3.1 カテゴリ一覧

| カテゴリ | 説明 |
|----------|------|
| STRATEGY | 思考・戦略性 |
| LUCK | 運要素 |
| INTERACTION | 対人干渉 |
| PARTY | 盛り上がり |
| SPEED | テンポ |
| ENGINE | エンジン構築 |
| SOCIAL | 会話・心理 |
| CONTROL | リソース管理 |

---

# 4. mechanics → 中間カテゴリ変換

## 4.1 定義例

```json
{
  "Worker Placement": {"STRATEGY": 3},
  "Deck Building": {"ENGINE": 3},
  "Dice Rolling": {"LUCK": 3},
  "Bluffing": {"SOCIAL": 3},
  "Area Control": {"INTERACTION": 3},
  "Real-Time": {"SPEED": 3},
  "Set Collection": {"CONTROL": 2}
}
```

## 4.2 方針
1 mechanic = 複数カテゴリへの加算も可
スコアは相対値（1〜3）で管理
未対応mechanicは無視してよい


# 5. 中間カテゴリ → Boardoryスコア変換

## 5.1 最終スコア軸
| 軸 | 意味 |
|----|-----|
| strategy | 戦略性 |
| luck | 運要素 |
| interaction | 対人性 |
| party | 盛り上がり |
| speed | テンポ |

## 5.2 変換テーブル

```json
{
  "STRATEGY": {"strategy": 3},
  "ENGINE": {"strategy": 2, "speed": -1},
  "LUCK": {"luck": 3},
  "INTERACTION": {"interaction": 3},
  "SOCIAL": {"interaction": 2, "party": 2},
  "PARTY": {"party": 3},
  "SPEED": {"speed": 3},
  "CONTROL": {"strategy": 2}
}
```


# 6. Weight（複雑さ）の反映

## 6.1 定義

BGGの weight はゲームの複雑さを表す（1〜5）

## 6.2 反映ロジック

strategy += weight × 2
speed -= weight × 2


# 7. カテゴリ補正（任意）

BGG categoryを補助的に利用

## 7.1 例

```json
{
  "Party Game": {"party": 3},
  "Economic": {"strategy": 2}
}
```


## 7.2 注意

フレーバー系カテゴリ（Fantasy等）は無視
性格に影響するもののみ使用


# 8. スコア計算

## 8.1 処理手順

- mechanics取得
- 中間カテゴリに変換
- カテゴリスコア合算
- Boardoryスコアへ変換
- weight補正
- category補正


# 9. タイプ分類

## 9.1 メインタイプ

最大スコアの軸で決定

| タイプ | 条件 |
|-----|------|
| ストラテジスト | strategy最大 |
| インタラクター | interaction最大 |
| パーティメーカー | party最大 |
| カジュアル | luck + speed高 |
| バランス型 | 全体平均 |


## 9.2 サブタイプ

2番目に高い軸と組み合わせ

例：

戦略 × インタラクション → 対戦ストラテジスト
パーティ × ソーシャル → 盛り上げ屋

## 10. データ構造

```json
{
  "scores": {
    "strategy": 70,
    "luck": 20,
    "interaction": 50,
    "party": 30,
    "speed": 10
  },
  "type": "ストラテジスト",
  "subType": "対戦ストラテジスト"
}

```
