# Boardory ボドゲタイプ診断 ロジック設計

## 概要

Boardoryは、ユーザーのプレイ記録（ゲームのBGGメカニクス・weight・プレイ回数・評価）から
3つの軸スコアを算出し、10種類のプレイスタイルタイプに分類する診断機能を持つ。

### 設計方針

- 機械学習は使用せず、完全ルールベースで実装する
- 精度より「納得感」と「一貫性」を優先する
- パラメータは JSON ファイルで管理し、コードを変更せずに調整可能にする

---

## 処理フロー

```text
プレイ記録（gameId × sessionCount × rating）
  ↓
BGGデータ参照（mechanics, weight）
  ↓
mechanics → 中間カテゴリ変換（MECHANICS_MAP）
  ↓
中間カテゴリ → 3軸スコアへ変換（INTERMEDIATE_TO_3AXES）
  ↓
BGG weight で Depth を補正（weight が主成分）
  ↓
sessionCount × ratingFactor で加重平均
  ↓
優先度順に条件マッチング → タイプ決定
```

## 1. 3軸スコア（Boardoryスコア）

| 軸 | 意味 | 低い側（0） | 高い側（100） |
| --- | --- | --- | --- |
| `depth` | ゲームの重さ・複雑さ | ライトゲーマー | ヘビーゲーマー |
| `competition` | 対戦性 | 協力型 | ガチ対戦型 |
| `chaos` | カオス度 | 戦略型（確定的） | 運ゲー型（ランダム） |

スコアはすべて **0〜100** の整数値。

---

## 2. 中間カテゴリ

BGGメカニクスを9種類の中間カテゴリに圧縮してから、3軸スコアに変換する。

| カテゴリ | 説明 |
| --- | --- |
| `STRATEGY` | 思考・戦略性（Worker Placement等） |
| `ENGINE` | エンジン・構築系（Deck Building等） |
| `CONTROL` | リソース管理・最適化 |
| `LUCK` | 運・ランダム性（Dice Rolling等） |
| `INTERACTION` | 対人干渉（Area Control, Take That等） |
| `SOCIAL` | 心理戦・コミュニケーション（Bluffing, Social Deduction等） |
| `COOP` | 協力・チーム（Cooperative Game, Team-Based Game） |
| `PARTY` | 盛り上がり系（Acting, Trivia等） |
| `SPEED` | リアルタイム・スピード系 |

`COOP` は `SOCIAL` から分離した専用カテゴリ。協力ゲームは competition 軸を下げる。

---

## 3. MECHANICS_MAP（`src/data/mechanics-map.json`）

BGGメカニクス文字列から中間カテゴリへの変換テーブル。

- 重みは相対値（1〜3）
- 1つのメカニクスが複数カテゴリに貢献してよい
- 未定義メカニクスは無視（エラーなし）

### 代表的なエントリ

```json
"Worker Placement":    { "STRATEGY": 3 }
"Deck Building":       { "ENGINE": 3 }
"Engine Building":     { "ENGINE": 3 }
"Resource Management": { "CONTROL": 3 }
"Dice Rolling":        { "LUCK": 3 }
"Push Your Luck":      { "LUCK": 3 }
"Area Control":        { "INTERACTION": 3 }
"Take That":           { "INTERACTION": 3 }
"Negotiation":         { "INTERACTION": 3, "SOCIAL": 1 }
"Cooperative Game":    { "COOP": 3 }
"Team-Based Game":     { "COOP": 3 }
"Bluffing":            { "SOCIAL": 3 }
"Hidden Roles":        { "SOCIAL": 3, "INTERACTION": 1 }
"Social Deduction":    { "SOCIAL": 3, "INTERACTION": 1 }
"Acting":              { "PARTY": 3, "SOCIAL": 1 }
"Real-Time":           { "SPEED": 3 }
"Speed Matching":      { "SPEED": 3 }
```

## 4. 中間カテゴリ → 3軸変換（`INTERMEDIATE_TO_3AXES`）

各カテゴリが3軸に与える影響度（正 = 軸を上げる、負 = 下げる）。
スケール係数 `AXIS_SCALE = 4` を掛けてからベースライン50に加算する。

```text
STRATEGY    → depth: +2, competition:  0, chaos: -2
ENGINE      → depth: +3, competition:  0, chaos: -1
LUCK        → depth:  0, competition:  0, chaos: +3
INTERACTION → depth:  0, competition: +3, chaos:  0
SOCIAL      → depth:  0, competition: +1, chaos:  0
COOP        → depth:  0, competition: -3, chaos:  0
PARTY       → depth:  0, competition:  0, chaos: +1
SPEED       → depth: -2, competition:  0, chaos:  0
CONTROL     → depth: +1, competition:  0, chaos: -1
```

`competition` のベースラインは50（中立）。INTERACTION が上げ、COOP が下げる。
`chaos` のベースラインは50。LUCK が上げ、STRATEGY/ENGINE/CONTROL が下げる。

---

## 5. Depth の計算（weight が主成分）

```typescript
const mechDepth = clamp(50 + depthContrib * AXIS_SCALE)

const depth = weight === null
  ? mechDepth
  : clamp((weight - 1) / 4 * 100 * 0.7 + mechDepth * 0.3)
```

- `weight` がある場合: weight 70% + メカニクス由来 30%
- `weight` がない場合: メカニクスのみで算出

| weight | depth 目安 |
| --- | --- |
| 1.0（極軽） | ≈ 15 |
| 2.0 | ≈ 33 |
| 3.0（中程度） | ≈ 50 |
| 4.0 | ≈ 68 |
| 5.0（極重） | ≈ 85 |

---

## 6. プレイ記録の重み付け

```typescript
entryWeight = sessionCount × ratingFactor(rating)
```

### ratingFactor

| 評価 | 倍率 |
| --- | --- |
| 未評価 | 1.0 |
| ★1 | 0.6 |
| ★2 | 0.8 |
| ★3 | 1.0 |
| ★4 | 1.2 |
| ★5 | 1.5 |

高評価のゲームはプロフィールへの影響が大きくなる。
複数ゲームの加重平均: `axisSum / totalWeight`。

## 7. カテゴリフォールバック

メカニクスが未マッピングかつ BGG weight もない場合、`categories` フィールドを使用して3軸スコアを直接割り当てる。

| カテゴリ | depth | competition | chaos |
| --- | --- | --- | --- |
| Abstract Strategy | 60 | 60 | 10 |
| Party Game | 10 | 50 | 60 |
| Economic | 60 | 65 | 30 |
| Wargame | 70 | 80 | 30 |
| Cooperative | 40 | 15 | 45 |
| Dice | 15 | 50 | 85 |
| ... | ... | ... | ... |

---

## 8. タイプ分類（優先度順マッチング）

先にマッチした条件が採用される。

```text
1. cooperative      : competition ≤ 28
2. gambler          : chaos ≥ 72
3. heavy-strategist : depth ≥ 60 AND competition ≥ 55 AND chaos ≤ 42
4. engine-builder   : depth ≥ 60 AND chaos ≤ 42
5. negotiator       : competition ≥ 68 AND chaos ≤ 45
6. strategic-player : depth ≥ 45 AND competition ≥ 60 AND chaos ≤ 45
7. trickster        : competition ≥ 62
8. party-master     : depth ≤ 40 AND chaos ≥ 38
9. casual           : depth ≤ 45 AND chaos ≤ 55
10. balanced        : 上記に当てはまらない（万能型）
```

## 10. 特殊ケース

### 入力なし（entries が空）

```typescript
{ id: "balanced", scores: { depth: 33, competition: 50, chaos: 50 } }
```

### メカニクスが全て未マッピング

カテゴリフォールバックも失敗した場合（weight・categories ともになし）、そのゲームはスコア計算から除外される。全ゲームが除外された場合は balanced を返す。
