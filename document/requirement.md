# Boardlog 要件定義書

## 1. 概要

### 1.1 サービス名

Boardlog（仮）

### 1.2 サービス概要

ユーザーが遊んだボードゲームを記録し、後から振り返ることができるサービス。
ボードゲームの箱画像を中心としたビジュアルで履歴を表示し、「思い出アルバム」のように体験を保存する。

### 1.3 コンセプト

ボードゲームの **思い出アルバム + 自分のボドゲ棚**

サービスの成長段階：

```
思い出記録
↓
ボドゲ履歴
↓
自分の本棚
↓
ボドゲ名刺（プロフィール）
```

---

# 2. 背景

ボードゲームのプレイ体験は以下のような特徴がある。

* 一度遊んだゲームを後から思い出したい
* 「あのゲーム面白かった」という体験を保存したい
* ボードゲーム会で「どんなゲームが好き？」という会話がよく発生する

既存サービスは **データ管理寄り**であり、体験の記録や可視化にフォーカスしたものは少ない。

---

# 3. 目的

本サービスの目的は以下。

1. ボードゲームのプレイ体験を記録する
2. 過去のプレイ履歴を視覚的に振り返る
3. 将来的にはユーザーの「ボドゲ棚」をプロフィールとして公開できるようにする

# 7. MVP機能要件

## 7.1 ゲーム検索

ユーザーはボードゲームを検索できる。

表示項目

* ゲーム名
* 箱画像

データソース

BGG API

一次ソース
https://boardgamegeek.com/wiki/page/BGG_XML_API2

---

## 7.2 プレイ記録

ユーザーは遊んだボードゲームを記録できる。

保存項目

| 項目       | 説明    |
| -------- | ----- |
| gameId   | ゲームID |
| playedAt | プレイ日  |
| rating   | 面白さ   |
| memo     | メモ    |

---

## 7.3 プレイ履歴一覧

ユーザーが遊んだゲームを一覧表示する。

UI

箱画像のグリッド

例

```
📦 📦 📦
📦 📦 📦
📦 📦
```

---

## 7.4 プレイ詳細表示

ゲームごとに記録を確認できる。

表示内容

* ゲーム名
* 箱画像
* プレイ日
* 評価
* メモ

---

# 8. 将来機能（Phase2以降）

## 8.1 ボドゲ棚

ユーザーの遊んだゲームを棚として表示。

例

```

📦カタン
📦ドミニオン
📦カルカソンヌ
📦テラフォーミングマーズ
```

---

## 8.2 ボドゲ名刺（プロフィール）

例

```
さほ

好きなゲーム
📦テラフォーミングマーズ
📦ブラス

遊んだゲーム
42種類
```

URL

```
board-log.pekori.dev/username
```

## 8.3 統計機能

例

* 遊んだゲーム数

* 好きなジャンル

---

## 8.4 SNS共有

共有先
